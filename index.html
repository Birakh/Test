📋 Field Guide
System — New vs Old:
New = newer employment system — Auto-Fill uses a flexible weekly hour target (alternating slightly higher/lower weeks around the contract hours) and includes 13H and 6H shifts in the mix.
Old = older employment system — Auto-Fill uses a strict rotating week pattern: one full week of Early shifts (8E), the next full week of Late shifts (8L/8M), and repeats. No 13H or 6H assigned. ⚠️ System has no effect on Immo workers (Fixed or Jump-in) — their schedule is always based on the Immo Role setting below.

Shift Preference (non-Immo workers only):
Mixed Shifts = Auto-Fill freely assigns 8E, 8L, 8M, 13H, and 6H to balance the schedule and hit daily targets. Best for most staff.
Early Only = only 8E shifts assigned — never a late or mid shift. Good for workers who can't do late evenings.
Late Only = only 8L/8M shifts assigned — never an early. Good for workers who prefer afternoons/evenings. Immo Role:
Standard = regular ward worker, no Immo involvement.
Fixed Immo = works in Immo only (ImE 07:00–15:30 or ImL 07:30–16:00). System and Shift Preference fields are ignored.
Immo Jump-in = normally a regular worker. Manually assign ImE/ImL in the table when Immo needs cover (sick, understaffed).

⚙️ Connection Settings

✔ PAT updated Create at GitHub → Settings → Developer settings → Personal access tokens → Fine-grained
Needs Contents: Read & Write permission on this repo. Stored only in your browser's localStorage.
📋 One-time setup:
1. Create (or use your existing) GitHub repo — the same one hosting the GitHub Pages site.
2. Each team member creates their own PAT with Contents: Read & Write permission, enters it here, and saves.
3. The planner saves shiftplanner_data.json and shiftplanner_config.json directly in the repo root.
4. Everyone accesses the same shared schedule via the GitHub Pages link. ✅
Pull = download the shared version from GitHub into your local session.
Push = upload your current session to GitHub so all colleagues see it.
⚠️ Last push wins — coordinate with your team or use auto-sync (which pulls before pushing).
🔐 Data Encryption Password

All JSON files are AES-encrypted with this shared team password. Every team member must enter the same password to read the schedule. Leave blank to disable encryption.
